# 指标说明（通用）

本文档用于快速查阅负载测试常见指标的含义与计算方式。
它是通用说明，不代表某个具体实现已支持全部指标。

---

* RPS (Requests Per Second) - 每秒请求数
  * 衡量目标：服务器在单位时间内接收到的所有请求的总数量。
  * 包含范围：无论是成功处理的请求、客户端错误（如 400、404）、服务器内部错误（如 500），还是超时未完成的请求，都会被计入 RPS。
  * 主要用途：衡量总负载能力与流量压力，反映外部请求的进入规模。

* QPS (Queries Per Second) - 每秒查询数（或每秒成功处理请求数）
  * 衡量目标：服务器在单位时间内成功响应并处理的请求总数量。
  * 包含范围：通常指返回 2xx 状态码的请求，不包含失败请求。
  * 主要用途：衡量实际有效处理能力与服务质量。

* TPS (Transactions Per Second) - 每秒事务数
  * 衡量目标：单位时间内完成的业务事务数量。
  * 包含范围：一笔事务可能包含多次请求；口径依赖业务定义。
  * 主要用途：衡量业务层面的处理能力。

* Arrival Rate - 注入速率（开放模型）
  * 衡量目标：调度器尝试发起请求的目标速率。
  * 包含范围：调度侧计划速率，不保证请求一定完成。
  * 主要用途：用于描述外部负载模型，与并发模型不同。

* Throughput / Bandwidth - 吞吐与带宽
  * 衡量目标：单位时间内传输的字节数。
  * 包含范围：可区分 Down（响应字节）与 Up（请求字节）。
  * 主要用途：衡量链路压力与数据传输能力。

* Latency (End-to-End) - 端到端延迟
  * 衡量目标：从请求发起到响应完成的总耗时。
  * 包含范围：包括网络传输、排队、处理与响应。
  * 主要用途：衡量整体体验与性能。

* TTFB (Time to First Byte) - 首字节时间
  * 衡量目标：从请求发起到收到首字节的时间。
  * 包含范围：包含连接建立、服务端处理与网络传输首包。
  * 主要用途：衡量服务端响应速度与链路首包延迟。

* Connect / DNS / TLS - 连接相关耗时
  * 衡量目标：连接建立的分阶段耗时。
  * 包含范围：DNS 解析、TCP 握手、TLS 握手等。
  * 主要用途：定位连接层瓶颈。

* P50 / P90 / P95 / P99 - 延迟分位数
  * 衡量目标：延迟分布在不同百分位的值。
  * 包含范围：基于请求延迟样本统计。
  * 主要用途：观察尾延迟与稳定性。

* Min / Max / StdDev - 最小值 / 最大值 / 标准差
  * 衡量目标：延迟或吞吐的波动范围。
  * 包含范围：基于样本统计。
  * 主要用途：衡量稳定性与抖动。

* Success Rate / Error Rate - 成功率 / 错误率
  * 衡量目标：成功与失败请求的比例。
  * 包含范围：成功通常指 2xx，失败包括 4xx/5xx/超时。
  * 主要用途：衡量稳定性与可用性。

* Status Code Distribution - 状态码分布
  * 衡量目标：按 HTTP 状态码聚合的计数分布。
  * 包含范围：2xx/3xx/4xx/5xx 等。
  * 主要用途：快速识别错误类型与占比。

* Timeouts / Retries - 超时 / 重试
  * 衡量目标：超时次数与重试次数。
  * 包含范围：由超时策略与重试策略决定。
  * 主要用途：衡量稳定性与异常恢复能力。

* Concurrency / Workers / Threads - 并发与执行单元
  * 衡量目标：并发执行的单位数量。
  * 包含范围：含义依实现而定（协程、线程或 worker）。
  * 主要用途：描述并发压力配置。

* In-flight Requests - 进行中的请求
  * 衡量目标：当前正在处理的请求数量。
  * 包含范围：已发出但未完成的请求。
  * 主要用途：衡量实时并发压力。

* Connections - 连接数与复用
  * 衡量目标：当前连接数量与复用情况。
  * 包含范围：短连接/长连接/HTTP2 多路复用等。
  * 主要用途：评估连接策略与资源占用。

* Total Bytes / Avg Bytes Per Request - 字节累计与均值
  * 衡量目标：累计字节与单请求平均字节。
  * 包含范围：可区分上行/下行。
  * 主要用途：衡量负载体量与传输成本。

* Ramp-up / Steady / Ramp-down - 阶段调度
  * 衡量目标：负载的升压、稳定、降压阶段。
  * 包含范围：阶段长度与目标速率。
  * 主要用途：模拟真实流量变化。

* Duration / Elapsed - 计划时长 / 已运行时间
  * 衡量目标：总持续时间与当前进度。
  * 包含范围：由测试计划与运行时钟决定。
  * 主要用途：监控测试进度与结果稳定性。

* Think Time / Pacing - 思考时间 / 节流
  * 衡量目标：请求间隔或节奏控制。
  * 包含范围：固定、随机或分布式延迟。
  * 主要用途：模拟真实用户节奏。
